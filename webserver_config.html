Webserver Config Tools

The re-design of 5 webserver config tools with a new and improved user experience.

Overview

The company's strategic business objective is to 'sunset KonsoleH'. KonsoleH is the company's old control panel which is being be replaced by the new xneelo Control Panel (xCP). The goal of this project was to re-design, with a new and improved user experience, 5 webserver config tools that are available in KonsoleH.

URL Redirects

What this tool does

This tool allows you to redirect your entire website or a specific web page to a different URL than the one originally requested. For example, if you've changed your business name and are migrating to a new domain, you can create a permanent redirect from your old website ro the new website URL. Another example is you might want to conduct A/B testing so you want to temporarily redirect visitors to another webpage.

Features for MVP

As part of MVP, the business determined that all functionality that is available in KonsoleH (the old control panel) has to be available for the new xneelo Control Panel, in order to sunset KonsoleH (SBO). Such functionality includes:

  • Add a redirect (301 Permanent, 302 Temporary, 401 Gone and 303 See Other).
  • Edit a redirect.
  • Delete a redirect.

Customer Pain Points in KonsoleH

I conducted interviews with 5 consultants from xneelo technical support and asked them what the customer pain points were when using this tool. This is what I found:

  • Customers get the URL syntax (http://) wrong of the redirection.
    • We solved this issue in the new design by adding a select component where the user can select the URL protocol before typing the URL path.
  • Customers get the file path wrong. They are not sure whether or not they have to add a forward slash /.
    • We solved this issue by implementing proper validation and adding adequate helper text below the text input.
  • Customers do not understand that the / represents the root folder.
    • We solved this issue by adding radio buttons so that the user can select 'Entire website' instead of typing a forward slash (/).
  • Redirects donâ€™t work on multiple domains that have a parent domain with redirects in the code. The support consultants have to exclude the multiple domains manually.
    • We were not able to solve this issue for MVP but our team will be looking at this post MVP. For MVP we added a notification banner, explaining this to the user.
This is what the current tool looks like in KonsoleH

The Redesign for xCP

I added a notification banner explaining redirects donâ€™t work on multiple domains that have a parent domain with redirects in the code. The user can dismiss this banner at any point.
I added an expandable card above the table explaining what this tool does and which types of redirects the user can create. Clicking on the close icon button collapses the card.
The table displays all of the redirects that the user has created for this domain. The user can add, edit or delete redirects they have created.

Add Redirect Feature

When the user clicks on the Add button, this full page modal is displayed. The user can choose which type of redirect they want to create, if they want to redirect their entire website or a specific directory and which URL they want to redirect to.
In KonsoleH, the user would have to type "/" if they wanted to redirect their entire website. Now, the user can select the "Entire website" radio button instead.
The user can choose to redirect a specific website directory, subdirectory or file by selecting that radio button option.
The helper text specifies the user must enter the path they want to redirect from, starting with a forward slash. The placeholder text shows an example of what the path could look like.
After the user clicks Add, we are able to do a back-end check and if the website already has a redirect, in that case we display this error toast.
After the user clicks Add, we are able to do a back-end check and if the specified path already has a redirect, in that case we display this inline error validation. The Add button will remain in disabled state until the form passes all front-end validation.
In order to solve one of the customer pain points, I included the URL protocol as a dropdown and made the most common protocol (https://) the default option.
If the user fills out the form, but accidentally clicks on 'Cancel' or closes the modal, then we prompt them with this modal. This is to ensure that the user is okay with discarding their changes, as they will not be saved.
This Success toast is disaplayed in the parent page after the user adds a redirect.

Edit Redirect Feature

If the user want to edit their redirect, after they click on 'Edit' (next to the redirect they want to edit), a full page modal opens displaying the saved form. The user can then make the necessary changes and click Save.
After saving the form, the user will see a success toast in the parent page, saying that their changes were saved.

Delete Redirect Feature

If the user wants to delete a redirect, they are prompted with this confirmation modal.
After deleting the redirect, a success toast is displayed in the parent page.

Index Manager

What this tool does

When someone visits your website directory, the web server looks for specific index files in a specific order in order to display a web page on your browser. This tool allows you to customise these index files and the order in which the web server looks for them. If no file is found and directory browsing is disabled, a 403 Forbidden error is displayed. If no file is found and directory browsing is enabled, then your browser will display a list your website folders and files through hypertext links.

Features for MVP

As part of MVP, the business determined that all functionality that is available in KonsoleH (the old control panel) has to be available for the new xneelo Control Panel, in order to sunset KonsoleH. Such functionality includes:

  • Edit index files.
  • Enable directory browsing.
  • Disable directory browsing.
This is what the current tool looks like in KonsoleH.

The Redesign for xCP

The user can clearly see they are in the Index Manager tool through the red and underlined tab in the page header. The two main functionalities of the tool (edit index pages and enable/ disable directory browsing) are separated by the two cards.
If the user has not customised their index files, then the tool will display the default files that the webserver looks for (see below Index files in the top card). Directory browsing is off by default, since it's not recommended to display your website files and folders.

Edit Index Files

If the user has not changed the webserver's default index files, then when they click Edit and this full page modal opens, they will see the Default index files radio button pre-selected.
If the user wants to choose which files the webserver should look for, then they can select that radio button option.
When the user selects Change default index files, then a chip input will appear. The user can type one or multiple file names. The webserver will look for the first file and if it does not exist, then the webserver will look for the second file, and so on until a web page is displayed.
After saving the form, the user will see a success toast in the parent page, saying that their changes were saved.

Enable Directory Browsing

The user can click on the on/ off toggle to enable and disable diirectory browsing. If the toggle is off and the user clicks on it, then they get prompted with the enable directory confirmation modal. The modal explains the consequences of enabling directory browsing.
After confirming their action, by clicking on the Enable button, the modal closes and a success toast is displayed.

Disable Directory Browsing

If the toggle is on and the user clicks on it, then they get prompted with the disable directory confirmation modal. The modal explains the consequences of disabling directory browsing.
After confirming their action, by clicking on the Disable button, the modal closes and a success toast is displayed.

Custom Error Pages

What this tool does

This tool allows users to personalise their website's error pages rather than using the server's default error pages, making their website look more professional and helpful to browsers.

Features for MVP

As part of MVP, the business determined that all functionality that is available in KonsoleH (the old control panel) has to be available for the new xneelo Control Panel, in order to sunset KonsoleH. Such functionality includes:

  • Edit 4 types of error pages by specifying a file location or URL.
  • Revert back to the webserver's error page.
This is what the current tool looks like in KonsoleH

The Redesign for xCP

The user will see a table with four types of error pages: 401 Unauthorised, 403 Forbidden, 404 Not Found, 500 Internal Server Error. If the user has customised any of these error pages, then the table will display the File Location or URL of the error page.
I added an expandable card above the table explaining what this tool does and which error pages they can customise. The user can also collapse the card by clicking on the close icon.
The user can click on the kebab next to the error page they want to customise. The kebab will display an Edit option.
After the user clicks on Edit, they will be prompted with a full page modal. The modal will display the type of error page they are about to customise (in this case 401 Unauthorised). If the user has not customised this error page before, the Default option will be selected by default, since the website is displaying the webserver's default error page.
If the user wants to customise their error page, they can do so by specifying a file location or URL. If they've selected the File radio option, then a text input will appear where they can type the exact file location of the webpage they want to display for this type of error.
If the user wants to customise their error page, they can do so by specifying a file location or URL. If they've selected the URL radio option, then a text input will appear where they can select the URL protocol and type the URL path of the webpage they want to display for this type of error.
If the user edits the form, but accidentally clicks on Cancel or closes the modal, then we prompt them with this modal. This is to ensure that the user is okay with discarding their changes, as they will not be saved.
After confirming their action, by clicking on the Save button, the modal closes and a success toast is displayed.

IP Blocker

What this tool does

This tool blocks IP addresses from accessing the user's website and displaying a 403 error message. This is useful when blocking bots or spammers.

Features for MVP

As part of MVP, the business determined that all functionality that is available in KonsoleH (the old control panel) has to be available for the new xneelo Control Panel, in order to sunset KonsoleH. Such functionality includes:

  • Block a single IP or IP range.
  • Block a domain.
  • Delete to unblock IP's.

When conducting user testing, these were the features that were requested by users:

  • Block by country.
  • Block IPs for an entire server instead of per domain.
  • Import list of IP's to block multiple at a time.

These features will be implemented post MVP.

Customer Pain Points in KonsoleH

I conducted interviews with 5 consultants from xneelo technical support and asked them what the customer pain points were when using this tool. This is what I found:

  • Customers cannot import a list of IPs, they can only add one at a time.
  • Dedicated Server customers want the ability to block an IP address across an entire dedicated server
  • People donâ€™t know which formats they can use to specify blocked IPs
This is what the current tool looks like in KonsoleH

The Redesign for xCP

When the user goes to the IP blocker tool, they will see a list of all the blocked IP's for their website.
I added an expandable card above the table explaining what this tool does. The user can also collapse the card by clicking on the close icon.
The user is able to delete blocked IPs at any point. The can delete a single row by clicking on the row kebab and then on the Delete option.

Delete IPs

Since deleting a blocked IP doesn't have serious consequences, we don't prompt the user with a confirmation modal. They will see that the corresponding row is being deleted through the spinner.
If the user selects one or more rows but they click on the row kebab to delete a blocked IP, then only that row will be deleted. The remaining rows (that were selected but not deleted) will remain selected.
If the user wants to bulk delete, then they would have to click Delete on the table header, in order for all the selected rows to be deleted.
In this example, the user has performed a bulk delete action (by clicking Delete on the table header). You can see the Delete button in the table header has changed to a spinner and all the rows that are being deleted also display a spinner.
After all rows have been deleted, then the table updates accordingly and a Success toast is displayed.

Add IP

The user can block as many IP addresses from accessing their website. They can do that by clicking on the Add button in the IP blocker table. This will open up this full page modal.
In KonsoleH (the old control panel), users are not aware of which formats they can enter, when blocking an IP or IP range. In the new design, you can see I included all the valid formats and examples that the user can enter.
The user can also enter a URL and the tool will convert it into an IP address.
After the user enters the IP, IP range or URL that they want to block and click Add, it gets added to the table and a success toast is displayed.

Protect Directory

What this tool does

The Protect Directory tool is used to restrict access to specified directories on your domain thereby allowing you to grant privileged access to multiple users, each having their own login details e.g. username and password. When a visitor browses to a protected directory, they will be prompted to provide a username and password. Only if the correct username and password combination is provided, will the webpage load.

Features for MVP

As part of MVP, the business determined that all functionality that is available in KonsoleH (the old control panel) has to be available for the new xneelo Control Panel, in order to sunset KonsoleH. Such functionality includes:

  • Add authorised users.
  • Delete authorised users.
  • Reset password for authorised users.

Customer Pain Points in KonsoleH

I conducted interviews with 5 consultants from xneelo technical support and asked them what the customer pain points were when using this tool. This is what I found:

  • Customers donâ€™t know what the symbols or icons represent.
  • Customers cannot protect their whole website directory at once, only sub-folders. So a lot of manual intervention needed for this request.
This is what the current tool looks like in KonsoleH

The Redesign for xCP

This tool only got to the wireframe stages since there was a business decision to make it part of the File Manager tool instead.

When accessing this tool, the user will see a table with their website's directory. They will be able to navigate through their website's files and folders by clickin the the folder name.
At any point, they will also see if each folder or file is password protected or not. They can also click on Manager Users if they want to view their authorised users for that specific folder or file.
In this example, the user has selected their public_html, and they are viewing all the folders inside public_html. They are able to go back to their home directory by clicking on the breadbrumb navigation.
In this example, the user is clicked on the Manage users button for folder_one inside public_html. This will take the user to another screen where they will be able to see a list of all the authorised users for folder_one.
From here, the user can add additional authorised users, delete authorised users, reset their password and protect / unprotect this directory.

Add Authorised User

The user can add as many authorised users. They can do that by clicking on the Add button in the manage users table. This will open up this full page modal, where they would have to enter a username and a password. The authorised user would need to enter the same info when trying to access this

Reset Password

The user can reset password for all authorised users that they have created. They can do that by clicking on Reset password next to the corresponding authorised user.
Clicking on Reset password will open up this full page modal, where the user would enter a new password and click Reset. This would reset the password, close the modal and display a Success toast.

Delete Authorised User

The user can delete any authorised users that they have created. They can do that by clicking on the Delete button next to the corresponding authorised user.
Clicking on Delete will open up this confirmation modal, stating that this user will no longer have access to this part of their website. If the user confirms this action by clicking on the Delete button, this would remove access to the no lonfer authorised user, close the modal and display a Success toast.

Protect Website Directory

The user can password protect and remove the password protection for their website or webpqge at any point. They can do that by clickin on the On and Off toggle next to 'Password protect this directory'.
If the toggle is Off and the user clicks on it in order to password protect this directory, they will get prompted with this confirmation modal.
If the toggle is On and the user clicks on it in order to remove password protection for this directory, they will get prompted with this confirmation modal.
